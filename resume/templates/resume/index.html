The tech stack for this website is:

  • Python
  • Django
  • SQLite
  • HTML5
  • CSS3

The resume is loaded into Django and then parsed into various models so it can be stored in the SQLite database. When the page is requested the information is retrieved from the SQLite database and sent to the rendered view where it is inserted via Django templating. 100% of the styling is vanilla CSS (though some features are quite new and may not be implemented in every browser). That's right, there is absolutely no JavaScript or external libraries being used for any of the styling.

How the resume is broken down...

The main resume model contains fields for:

  • Name
  • Job Title
  • Email Address
  • Phone Number
  • Location
  • Blurb
  • ID

Then there is a model for job which includes:

  • Start Date
  • End Date
  • Job Title
  • Employer
  • Resume object it is associated with (Many-to-Many relationship)

Next we have the model for job bullet points. Separating them from jobs allows more flexibility, though they only have a few fields:

  • Text (Just whatever the bulletpoint is)
  • The job object it is associated with. A Foreign Key

The model for Skill includes fields for:

  • Category (to group the skills thematically)
  • Skill Name
  • Resume object (many to many relationship, a skill doesn't necessarily apply to only one resume and a resume can have multiple skills)

Social Media

  • Name
  • Link
  • Resume (m2m)

Lastly we have a model for "active resume" with only a foreign key field connecting it to a resume object.
